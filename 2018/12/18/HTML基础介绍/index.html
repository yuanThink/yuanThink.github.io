HTML、XHTML、HTML5 介绍

一、HTML简介及知识梳理

1、HTML概述

HTML (HyperText Markup Language) 不是一种编程语言;它是一种标记语言，用于告诉浏览器如何构造用户访问的网页。HTML由一系列的 元素组成, 可以使用它来封装，包装或标记内容的不同部分，使其以某种方式显示，或以某种方式执行。

注意： HTML对标签的大小写并不敏感，但是为了便于其他人阅读，以及辨认，最小全部写成小写的格式。

2、HTML发展史
版本 时间
HTML 1991
HTML+ 在1993年6月作为互联网工程工作小组（IETF）工作草案发布（并非标准）
HTML 2.0 1995年11月作为RFC 1866发布，在RFC 2854于2000年6月发布之后被宣布已经过时
HTML 3.2 1997年1月14日，W3C推荐标准
HTML 4.0 1997年12月18日，W3C推荐标准
HTML 4.01（ 微小改进 ） 1999年12月24日，W3C推荐标准
XHTML 1.0 2000年发布，是对HTML 4.01的一种扩展，语法更为严格
HTML 5 2012年12月17日，W3C正式宣布HTML5正式定稿；2014年10月29日HTML5标准规范制定完成，并已经完成了公开发布
XHTML 5 2013年发布，但并非是对HTML 5的一种扩展，而是“ 以XML序列化的HTML5 ”
3、HTML基础性页面结构
4、HTML基础性页面介绍
(1)<!DOCTYPE html>

声明文档类型解析方式：告诉浏览器当前的页面将以什么样的DTD(文档类型定义)去解析，常见的文档类型有：HTML5、HTML4.01、XHTML1.0等等，具体的文档类型内容请点击[doctype参考手册][http://www.w3school.com.cn/tags/tag_doctype.asp]

(2)<html></html>

此标签为根元素（标签），用来定义当前为html页面，里面包含了页面的所有内容。

(3)<head></head>

<head>标签用于定义文档的头部，它是所有头部元素的容器。<head>中的元素可以引用脚本、指示浏览器在哪里找到样式表、提供元信息等等。 ，比如：

  • meta： 元数据标签，此标签为单标签，常见的属性有：

    • name：指定了meta 元素的类型; 说明该元素包含了什么类型的信息。 常见的值有：
      • author：定义网页作者
      • description：为网页定义描述内容
      • Keywords ：为搜索引擎定义关键词，未来将会被淘汰，原因：有些网站拥有者，可能会大量填写关键词，导致搜索的结果未必是用户想要的结果
      • viewport：是用户网页的可视区域，手机浏览器是把页面放在一个虚拟的”窗口”（viewport）中，通常这个虚拟的”窗口”（viewport）比屏幕宽，这样就不用把每个网页挤到很小的窗口中（这样会破坏没有针对手机浏览器优化的网页的布局），用户可以通过平移和缩放来看网页的不同部分
    • charset ： 以什么编码格式进行解析网页

    • http-equiv：相当于文件的头作用，用于向浏览器传递一些有用的信息，以帮助正确和精确地显示网页内容，与之对应的属性为content

    • content： 对于name属性值的一种解释，倘若name为viewport则content必须设置以下的值：

      • width：控制 viewport 的大小，可以指定的一个值，如 600，或者特殊的值，如 device-width 为设备的宽度（单位为缩放为 100% 时的 CSS 的像素）
      • height：和 width 相对应，指定高度
      • initial-scale：初始缩放比例，也即是当页面第一次 load 的时候缩放比例
      • maximum-scale：允许用户缩放到的最大比例
      • minimum-scale：允许用户缩放到的最小比例
      • user-scalable：用户是否可以手动缩放
  • title：网页在浏览器中的标题，为闭合标签
  • link：单标签，其作用为：
    • 为网页添加自定义图标，常用的为.ico结尾的，单大多数浏览器也支持：.png|.gif
    • 为网页添加外联样式
  • style：网页内联样式表，为闭合标签
  • script：网页引入的JavaScript脚本语言，为闭合标签
(4)<body></body>

此为页面内容标签，即网页中99%的内容都是在此闭合标签中显示的，它包含了整个网页的页面布局以部分行为动作

7、HTML元素的分类

HTML元素的种类一般分为三种，第四种可忽略，具体如下：

  • 内联元素（行内元素）：内联元素在显示时通常不会以新行开始，并且利用css不可以更改其宽高，内联元素主要有：span、b、a、i、td等等
  • 块级元素：块级元素在浏览器显示时，通常会以新行来开始（和结束），利用css可修改器宽高，块级元素主要有：div、p、ul、ol、table等等
  • 内联块元素：同行显示并可修改宽高内外边距等属性，目前只有两种：img、input
  • 可变元素：根据上下文关系确定该元素是块元素还是内联元素，主要有：applet，button，del，iframe，ins ，map，object，script

二、XHTML简介及知识梳理

1、XHTML概述
（1）XHTML是什么？
  • XHTML 指的是可扩展超文本标记语言
  • XHTML 与 HTML 4.01 几乎是相同的
  • HTML 是更严格更纯净的 HTML 版本
  • XHTML 是以 XML 应用的方式定义的 HTML
（2）发布时间
  • XHTML 是 2001 年 1 月发布的 W3C 推荐标准
（3）浏览器支持情况
  • XHTML 得到所有主流浏览器的支持
2、为何要使用XHTML？

现如今，浏览器所支持的HTML文档类型比较众多，以及好多开发人员是非前端人员，开发的页面往往会丢弃的文档类型的声明，显得开发的网页比较混乱；以及最重要的是智能化时代的到来，PC端虽能够对大多数这些混乱的网页进行正确的解析，但是，在移动端却是收效胜微，往往不能正确的解析，因此，通过结合 XML 和 HTML 的长处，开发出了 XHTML。XHTML 是作为 XML 被重新设计的 HTML。

3、XHTML与HTML的区别
（1）文档结构
  • XHTML页面的DOCTYPE 是强制性的(即必须存在)
  • <html> 中的 XML namespace 属性是强制性的(即必须存在)
  • <html>、<head>、<title>以及 <body> 也是强制性的(即必须存在)
（2）元素语法
  • XHTML 元素必须正确嵌套
  • XHTML 元素必须始终关闭
  • XHTML 元素必须小写
  • XHTML 文档必须有一个根元素
（3）属性语法
  • XHTML 属性必须使用小写
  • XHTML 属性值必须用引号包围
  • XHTML 属性最小化也是禁止的
补充说明：
  • XHTML 元素必须合理嵌套
  • XHTML 元素必须有关闭标签
  • 空元素必须包含关闭标签
  • XHTML 元素必须是小写
  • 属性名称必须是小写
  • 属性值必须有引号
  • 不允许属性简写

三、HTML5简介以及知识梳理

1、什么是HTML5？

HTML5 是下一代 HTML 标准 。

2、HTML5的发展史

HTML5 是 W3C 与 WHATWG 合作的结果，WHATWG 指 Web Hypertext Application Technology Working Group。

WHATWG 致力于 web 表单和应用程序，而 W3C 专注于 XHTML 2.0。在 2006 年，双方决定进行合作，来创建一个新版本的 HTML。

为 HTML5 建立的一些规则：

  • 新特性应该基于 HTML、CSS、DOM 以及 JavaScript。
  • 减少对外部插件的需求（比如 Flash）
  • 更优秀的错误处理
  • 更多取代脚本的标记
  • HTML5 应该独立于设备
  • 开发进程应对公众透明
